# Risk Assessment Rule Management System - Solution Overview

## Date: October 22, 2025

---

## EXECUTIVE SUMMARY

This solution is an AI-powered risk assessment system that learns from human expertise to help organizations standardize and improve their risk management processes. It consists of three main components working together to create, refine, and apply risk assessment rules.

---

## WHAT THIS SOLUTION DOES

### 1. RULE GENERATION AND LEARNING (Jupyter Notebook: try11hack25_Works.ipynb)

**Purpose:** Learn from human risk assessment expertise to create automated rules

**Process:**
- Loads survey data from a spreadsheet (e.g., "250905 Risk Heuristics survey download.xlsx")
- Each column represents a question, and rows contain expert responses
- Uses Anthropic's Claude AI (claude-sonnet-4-5-20250929) to analyze responses and extract implicit reasoning patterns
- Generates two types of rules:
  - JSON format: Structured rules with descriptions and validation criteria
  - Plain text format: Human-readable rules for documentation
- Saves generated rules to `generated_rules.json` and `generated_rules.txt`

**Deduplication and Merging:**
- Applies fuzzy string matching (using fuzzywuzzy library) to identify similar rules
- Groups rules by common keywords to avoid redundancy
- Uses LLM-based semantic merging to combine similar rules into clearer, more effective versions
- Produces multiple output files:
  - `unique_rules_strict.json` - Rules deduplicated with 98% similarity threshold
  - `llm_merged_rules_keywords.json` - Final merged rules grouped by common keywords
  - `unique_rules_by_desc.json` - Rules grouped by description similarity

**Data Splitting for Model Validation:**
- Splits the final rule set into three datasets using scikit-learn (70/15/15 split):
  - `rules_training.json` (70%) - For training/tuning the classification model
  - `rules_testing.json` (15%) - For testing model performance
  - `rules_validation.json` (15%) - For final validation

**Key Features:**
- Converts tacit human knowledge into explicit, reusable rules
- Reduces rule duplication through intelligent merging
- Maintains traceability from survey responses to final rules
- Supports both structured (JSON) and readable (text) formats

### 2. USER RISK ENTRY AGENT (User_risk_agent.py)

**Purpose:** Guide users in creating high-quality risk and mitigation entries

**Target Users:** Risk managers, project managers, and team members who need to document risks

**How It Works:**

1. **Initialization:**
   - Prompts user to select a rules file (e.g., `generated_rules.json`)
   - Loads the rules into memory for real-time classification

2. **Risk Entry Process:**
   - User enters a risk description
   - Agent classifies the entry quality:
     - **Good:** Matches 50% or more of rules (minimum 2 rules)
     - **OK:** Matches 1+ rules but less than 50%
     - **Poor:** Matches no rules
   - Displays matched rules and their validation criteria
   - Provides immediate feedback on which rules the entry satisfies

3. **Intelligent Help System:**
   - If entry is classified as "poor":
     - Provides specific tips for improvement
     - Suggests a template based on the best-matching rule
     - Shows the rule's description and criteria to guide the user
   - Helps users understand what makes a good risk entry

4. **Mitigation Entry:**
   - After risk entry, prompts for mitigation description
   - Applies same classification and feedback process
   - Ensures mitigations are as well-formed as risks

5. **Data Collection:**
   - Saves all entries to an Excel file: `DDMMYY_new_risks.xlsx`
   - Includes risk descriptions, mitigations, and classifications
   - Creates a structured dataset for analysis and reporting

**Key Features:**
- Real-time quality assessment using keyword matching
- Progressive guidance to improve entry quality
- Encourages consistency across risk documentation
- Creates timestamped audit trail of new risks
- Interactive and user-friendly command-line interface

**Classification Algorithm:**
- Extracts keywords from rule validation criteria (words >2 characters)
- Checks if keywords appear in user's entry
- Counts how many rules are matched
- Applies threshold-based classification logic

### 3. SME RULE REVIEWER AGENT (sme_rule_reviewer.py / sme_rule_reviewer_cli.py)

**Purpose:** Allow risk subject matter experts (SMEs) to review, refine, and maintain the rule set

**Target Users:** Risk SMEs, risk managers, and governance teams responsible for rule quality

**Capabilities:**

1. **Rule Viewing and Filtering:**
   - List all rules with pagination (10 rules at a time)
   - Filter rules by keyword (e.g., "trigger", "likelihood", "impact")
   - Display rule descriptions, validation criteria, and IDs
   - Navigate through large rule sets efficiently

2. **Rule Editing:**
   - Edit rule descriptions to improve clarity
   - Modify validation criteria to be more specific
   - Edit both fields simultaneously
   - Preview current values before making changes

3. **Rule Management:**
   - Add new rules as patterns emerge
   - Delete outdated or incorrect rules
   - Confirmation prompts prevent accidental deletions

4. **AI-Powered Improvement:**
   - Uses Claude AI to analyze individual rules
   - Provides suggestions for:
     - Improving clarity and readability
     - Making criteria more specific and measurable
     - Identifying gaps or ambiguities
   - Extracts and applies AI suggestions with one command
   - Rationale provided for each suggestion

5. **Intelligent Search:**
   - Natural language search powered by Claude AI
   - Example: "search rules about likelihood assessment"
   - Returns relevant rule numbers with explanations
   - Searches descriptions and criteria semantically, not just keywords

6. **Change Tracking:**
   - Records all modifications with timestamps
   - Shows what was changed (field, old value, new value)
   - Tracks additions, deletions, and AI improvements
   - View full change history before saving

7. **Safe Data Management:**
   - Automatic timestamped backups before saving
   - Backup files named: `[filename].backup_YYYYMMDD_HHMMSS`
   - Prompts before exiting with unsaved changes
   - Separate save and save-quit commands

**Available Commands:**
- `list [start_index]` - View rules with pagination
- `filter <keyword>` - Find rules containing specific terms
- `edit <number>` - Modify a specific rule
- `add` - Create a new rule
- `delete <number>` - Remove a rule
- `suggest <number>` - Get AI improvement suggestions
- `search <query>` - Natural language search
- `changes` - View session modifications
- `save` - Save changes with backup
- `quit` - Exit (prompts if unsaved changes)
- `save-quit` - Save and exit

**Key Features:**
- Terminal-based interface (runs in VS Code or any CLI)
- No special dependencies beyond Python and Anthropic API
- Maintains rule quality over time
- Enables continuous improvement of the rule set
- Supports collaborative rule governance

## SYSTEM WORKFLOW

### Phase 1: Learning from Experts
1. Collect survey responses from risk experts
2. Run Jupyter notebook to generate rules from responses
3. AI extracts patterns and reasoning from human responses
4. Fuzzy matching identifies and merges similar rules
5. Keyword-based grouping reduces redundancy
6. Final rule set saved to JSON

### Phase 2: Rule Refinement (Optional but Recommended)
1. SME runs `sme_rule_reviewer_cli.py`
2. Reviews generated rules for clarity and accuracy
3. Uses AI suggestions to improve rule quality
4. Edits, adds, or removes rules as needed
5. Saves refined rule set with automatic backup

### Phase 3: User Application
1. User runs `User_risk_agent.py`
2. Selects the refined rule set file
3. Enters risk and mitigation descriptions
4. Receives real-time feedback and classification
5. Improves entries based on guidance
6. All entries saved to Excel for reporting

### Phase 4: Continuous Improvement
1. Periodically review new risk entries
2. Identify patterns in "poor" classifications
3. Use SME agent to add new rules or refine existing ones
4. Update rule set and redistribute to users
5. Monitor improvement in entry quality over time

## USE CASE EXAMPLES

### Scenario 1: Initial Setup
A risk manager wants to standardize how their team documents risks.
1. They distribute a survey asking experts to describe good vs. poor risk descriptions
2. Run the Jupyter notebook to generate 50+ rules from 20 survey responses
3. Use SME agent to review and refine the rules to 30 high-quality rules
4. Distribute the rule file and User_risk_agent.py to the team
5. Team members now receive consistent guidance when entering risks

### Scenario 2: Ongoing Maintenance
After 3 months, the risk manager notices many "poor" classifications for cyber risks.
1. Open SME agent and search for "cyber security rules"
2. Notice only 2 rules cover cyber risks
3. Add 5 new rules specific to cyber risk descriptions
4. Use AI suggestions to make the rules more specific
5. Save and redistribute updated rule file
6. Team's cyber risk entries improve immediately

### Scenario 3: Quality Improvement
A user struggles to get "good" classification for their risk entry.
1. User enters: "The project might have issues"
2. Agent classifies as "poor" (0 matched rules)
3. Agent suggests a template: "Risk: [specific threat] could cause [specific impact] due to [root cause]"
4. User revises: "Budget overrun risk could cause 20% cost increase due to scope creep from unclear requirements"
5. Agent classifies as "good" (matched 4 rules)
6. User receives positive feedback and learns what makes a good entry

## LIMITATIONS AND CONSIDERATIONS

### Scalability
- File-based storage limits concurrent usage
- Large rule sets (>500 rules) may slow down classification
- No database backend for enterprise-scale deployments

### Rule Quality
- Initial rule quality depends on survey response quality
- AI-generated rules require SME review before use
- Rules may reflect biases in survey respondents

### User Experience
- Command-line interface may not suit all users
- No graphical user interface (GUI)
- Learning curve for terminal commands

### AI Dependencies
- Requires internet connection for AI features
- Anthropic API usage incurs costs
- AI suggestions require human review (not automatically applied)

### Data Privacy
- All data stored locally in JSON/Excel files
- No encryption of files by default
- No audit trail of file access

## CONCLUSION

This solution provides a **foundation for standardizing risk documentation** by capturing expert knowledge as rules and guiding users to create high-quality entries. It excels at:

✅ Learning from human expertise
✅ Providing real-time feedback to users
✅ Enabling SMEs to maintain rule quality
✅ Creating consistent risk documentation
✅ Reducing subjectivity in risk descriptions

However, it is **not a complete risk management system**. It focuses specifically on improving the quality of risk and mitigation descriptions through AI-powered guidance and rule-based classification.

Organizations should view this as a **complementary tool** that enhances existing risk management processes, not a replacement for comprehensive risk management platforms.
